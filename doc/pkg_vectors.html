Vectors Package

A collection of common routines for working with dynamic arrays (indexed lists). This is based on the work of Grady Booch.

Introduced: SparForte 2.5

GCC Ada Equivalent: Ada.Containers.Vectors

This package includes new generic types:

  • vectors.cursor( key type, element type ) - a cursor for iterating through a vector
  • vectors.vector( key type, element type ) - a dynamic array

Ada: Because AdaScript does not have subtypes with ranges, you cannot change the first index or last index. For example, any type based on positive will start at index 1.

vectors.append( m, k, e )

 

Append string element e to the value under key k in the hash map. If the key does not exist, raise an exception.

Example

vectors.insert( fruit_unit, "apple", bushel" );
vectors.append( fruit_unit, "apple", "s" ); -- changed to "bushels"

Parameters
Param Mode Type Default Description
m in out hashed_maps.map required the map
k in map key type required the key
e in map element type required the string to append

Exceptions

An exception is raised if the key is the wrong type
An exception is raised if the key does not exist
An exception is raised if out of memory
An exception is raised if the element type is not a string

Restrictions

Not allowed with pragma ada_95

See Also

hashed_maps.prepend

Compare With

Implementation Note

This is in accordance with memcached.

vectors.append_elements( v, e [,c])

 

Queue list element e at the end of the vector v. If c is used, append c copies of e.

Example

vectors.append_elements( v, "zebra", 2 ); -- append 2 copies of "zebra"

Parameters
Param Mode Type Default Description
v in out vectors.vector required the vector
e in list element type required the element to queue
c in containers.count_type 1 the number of copies

Exceptions

An exception is raised if the copies is not an integer value
An exception is raised if out of memory

Compare With

Ada: Ada.Containers.Vectors.Append

Implementation Note

Ada's append is separated into individual subprograms due to SparForte's current lack of named parameters to differentiate the variations.

vectors.delete( v, c | i [, n] )

 

Remove an element from the vector at the index position, effectively sliding the elements toward the start of the vector by one position. If a cursor is given instead of an index, delete the item at the cursor position. If a count of n is given, delete n elements instead of one.

Example

-- if char_vector contains "a", "b", "c"
vectors.delete( char_vector, 2 ); -- vector is "a", "c"

Parameters
Param Mode Type Default Description
v in out vectors.vector required the vector
i in map key type required (if no cursor) the position to delete
c in vectors.cursor required (if no index) the cursor position to delete
n in containers.count_type 1 the number of elements to delete

Exceptions

An exception is raised if the number of elements is not an integer-type value

See Also

hashed_maps.reset
hashed_maps.set
